## Описание проекта
Grade Clicker — это Android-приложение, реализованное с использованием Jetpack Compose. Приложение представляет собой кликер, в котором пользователь набирает баллы, нажимая на изображение. В зависимости от количества баллов изменяются оценка, интерфейс и визуальные элементы.

> [!NOTE]
> Приложение демонстрирует работу жизненного цикла Activity и сохранение состояния при изменении конфигурации устройства.

---

## Цель работы
Цель работы: изучить жизненный цикл Activity в Android и научиться правильно
обрабатывать переходы между состояниями. В результате лабораторной работы будет
создано интерактивное приложение "Grade Clicker" (Кликер оценок), которое
демонстрирует работу lifecycle callbacks и сохранение состояния при изменении
конфигурации устройства.

## Контрольные вопросы

### Какие три метода вызываются при запуске приложения?
- `onCreate()`
- `onStart()`
- `onResume()`

### Чем отличается onPause() от onStop()?
- `onStop()` вызывается, когда Activity полностью становится невидимой

### Когда вызывается onRestart()?
Метод `onRestart()` вызывается при возврате к Activity после состояния `onStop()`

### Что происходит при повороте экрана?
Activity уничтожается (`onDestroy()`), затем создается заново (`onCreate()`), из-за чего состояние может быть потеряно

### В чем разница между remember и rememberSaveable?
- `rememberSaveable` сохраняет состояние даже при пересоздании Activity (например, при повороте экрана)

### Почему onCreate() должен вызывать super.onCreate()?
Потому что базовая реализация выполняет необходимую инициализацию Activity. Без этого приложение может работать некорректно или упасть

### Для чего используется класс Log?
Класс Log используется для вывода сообщений в Logcat, что помогает отслеживать выполнение программы и отлаживать приложение

### Какой метод вызывается только один раз за время жизни Activity?
`onCreate()`
